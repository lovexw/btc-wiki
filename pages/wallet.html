比特币钱包技术

1. 钱包概述

比特币钱包不存储比特币！比特币存在于区块链上。钱包只是管理私钥的工具。

通俗理解

钱包 = 钥匙串

  • 私钥 = 钥匙（用于花费比特币）
  • 地址 = 保险箱编号（用于接收比特币）
  • 余额 = 所有保险箱中的金额总和

丢失钱包=丢失钥匙，比特币还在区块链上，但你无法使用。

2. 钱包类型

2.1 按连网方式分类

① 热钱包（Hot Wallet）

  • 连接互联网的钱包
  • 便于使用
  • 安全性较低
  • 适合小额日常使用

② 冷钱包（Cold Wallet）

  • 离线存储私钥
  • 安全性高
  • 使用不便
  • 适合长期大额存储

2.2 按实现方式分类

① 软件钱包

  • 桌面钱包：Bitcoin Core、Electrum
  • 手机钱包：BlueWallet、Muun、Blockstream Green
  • 网页钱包：blockchain.com（不推荐，第三方控制私钥）

② 硬件钱包

  • 专用硬件设备
  • 私钥永不离开设备
  • 最安全的方案
  • 常见品牌：Ledger、Trezor、Coldcard

③ 纸钱包

  • 私钥打印在纸上
  • 完全离线
  • 易损坏、难备份
  • 已逐渐被淘汰

3. 分层确定性钱包（HD Wallet）

3.1 为什么需要HD钱包？

早期钱包为每个地址生成随机私钥：

问题：
- 需要备份每个私钥
- 生成100个地址 = 备份100次
- 容易遗漏

HD钱包从单一种子派生所有密钥：

优势：
- 只需备份一次（助记词）
- 可生成无限地址
- 有组织结构

3.2 BIP32/BIP44标准

派生路径：m/purpose'/coin_type'/account'/change/address_index

示例：m/44'/0'/0'/0/0
├─ 44' - BIP44标准
├─ 0' - 比特币（60'=以太坊）
├─ 0' - 账户0
├─ 0 - 外部链（接收地址）
└─ 0 - 地址索引0

m/44'/0'/0'/0/1 - 第二个接收地址
m/44'/0'/0'/1/0 - 第一个找零地址

3.3 助记词（BIP39）

12词助记词示例：
army van defense carry jealous true
garbage claim echo media make crunch

24词更安全（熵更大）

助记词 → 种子（512位）→ 主密钥 → 派生所有子密钥

4. 钱包功能

4.1 密钥管理

  • 生成密钥对
  • 安全存储私钥（加密）
  • 派生新地址
  • 导入/导出密钥

4.2 交易构建

  • 选择UTXO
  • 计算手续费
  • 创建找零输出
  • 签名交易
  • 广播到网络

4.3 余额查询

  • 扫描区块链
  • 识别属于自己的UTXO
  • 计算总余额
  • 追踪交易历史

5. 不同钱包类型详解

5.1 全节点钱包

Bitcoin Core

  • 下载完整区块链（500GB+）
  • 完全独立验证
  • 最高隐私和安全
  • 初始同步慢（数天）

5.2 SPV钱包

Electrum、Bluewallet

  • 只下载区块头（70MB）
  • 向服务器查询交易
  • 快速轻量
  • 隐私性略低

5.3 硬件钱包

工作原理

1. 私钥生成并存储在硬件中
2. 计算机创建未签名交易
3. 发送到硬件钱包
4. 用户在硬件设备上确认
5. 硬件钱包签名（私钥不离开设备）
6. 签名后的交易返回计算机
7. 计算机广播交易

即使电脑被黑，私钥也是安全的

6. 钱包安全最佳实践

安全建议

  • 备份助记词：抄写在纸上，多地存储
  • 永不截图：不要数字化存储助记词
  • 验证接收地址：防止恶意软件篡改
  • 使用硬件钱包：大额资金必须用硬件钱包
  • 测试恢复：定期用助记词测试恢复
  • 使用Passphrase：额外密码保护
  • 分散存储：不要all-in单一钱包
  • 警惕钓鱼：下载官方版本钱包

7. Passphrase（第25个词）

在助记词基础上添加额外密码：

相同助记词 + 不同Passphrase = 完全不同的钱包

优势：
- 增加安全层
- 防止物理胁迫（诱导钱包）
- 可以创建多个钱包

示例：
助记词 + "" → 钱包A（小额）
助记词 + "my-secret-phrase" → 钱包B（大额）

即使助记词泄露，没有Passphrase也无法访问钱包B

8. 多签钱包

需要多个密钥共同签名：

2-of-3多签方案：
- 密钥1：手机钱包
- 密钥2：硬件钱包
- 密钥3：纸钱包（备份）

任意两个即可花费
单一密钥丢失或被盗都不会损失资金

适用场景：
- 个人大额存储
- 公司资金管理
- 继承规划

9. 钱包互操作性

9.1 BIP标准

  • BIP32：HD钱包
  • BIP39：助记词
  • BIP44：多币种派生路径
  • BIP49：SegWit P2SH派生
  • BIP84：原生SegWit派生

遵循标准的钱包可以互相导入助记词恢复资金。

10. 地址生成策略

10.1 地址重用问题

永远不要重用地址！

  • 降低隐私（所有交易可关联）
  • 降低安全（多次暴露公钥）
  • 便于追踪

10.2 Gap Limit

钱包扫描地址的规则：

连续派生地址，如果连续20个地址没有交易，停止扫描

示例：
地址0-10有交易
地址11-30空
→ 只扫描到地址30

如果手动跳到地址50使用，钱包可能找不到！
解决：增加gap limit或按顺序使用地址

11. 常见钱包推荐

钱包 类型 特点 适合人群
Bitcoin Core 全节点 最高安全和隐私 技术用户
Electrum SPV 轻量、功能强大 日常使用
BlueWallet 移动SPV 简洁易用 初学者
Ledger 硬件 最安全 大额存储
Coldcard 硬件 完全离线 极致安全

12. 总结

核心要点

  • 钱包是密钥管理器：不存储比特币
  • HD钱包：单一备份，无限地址
  • 助记词：12/24词恢复所有密钥
  • 硬件钱包：大额必备
  • 备份至关重要：丢失=永久丢失
  • 隐私保护：不重用地址
